The task spec said to change only the `inputs` paths from `../kinds/...` to `/kinds/...`. During verification, `tools:validate-kinds` continued to fail with "Script not found 'validate-kinds.ts'" even after the inputs fix. Root cause: moon's toolchain-pinned bun (1.1.42 per `.moon/toolchain.yml`) treats `bun run validate-kinds.ts` as a script name lookup against package.json. This is a second pre-existing defect that the original plan did not account for.
